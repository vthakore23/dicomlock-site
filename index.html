The Problem

The file is the attack surface

A DICOM file isn't a picture. It is input for fragile parsers, full of fields an attacker controls. Click a region to see where.

Select a region above Each part of the format is something old hospital software has to parse, decode, and trust.

Weaponized files

The 128-byte preamble can carry an executable header, making one file both a valid scan and working malware. CVE-2019-11687 proved it; ELFDICOM extended it to Linux devices. Antivirus skips medical imagery.

Praetorian, 2025

Parsers that can't say no

Malformed length fields and deep nesting crash or hang imaging software. We reproduced a 140-byte file that demands 4 GB, a 30-million-times amplification. This is the GDCM and MicroDicom CVE class.

GDCM / MicroDicom, 2025

Vulnerable codecs everywhere

Compressed and video DICOMs decode through libjpeg, OpenJPEG, and FFmpeg-class libraries with long CVE histories, running deep inside the PACS on devices you cannot patch.

NVD / CISA, 2025
Claude Mythos · Project Glasswing

Anthropic's Mythos model found more than 10,000 high-severity bugs that humans and fuzzers missed for years, including a 27-year-old flaw in OpenBSD and 271 vulnerabilities in Firefox in a single pass. Over 99% are still unpatched. Glasswing gave about a dozen technology partners early access to patch first. Healthcare was left out (Health-ISAC, 2026), and it runs the most unpatchable legacy and embedded devices. DicomLock neutralizes the file before it reaches the flaw.

The Product

A security checkpoint in front of your PACS

DicomLock inspects every file before vulnerable software touches it. The scanner and the disarm/reconstruction engine are built and tested; a crypto-posture audit is in development.

01

Ingest

A file arrives from a scanner, outside facility, patient CD, or upload.

02

Scan

Polyglots, length-field attacks, nesting bombs, and codec-CVE exposure are all checked.

03

Disarm

The file is rebuilt from a validated canonical form. The image stays clinically identical.

04

Deliver

Clean files pass to PACS. Dangerous files are quarantined with a report.

Polyglot and payload detection Preamble and private tags scanned for executables and embedded payloads.
Structural integrity Every tag, VR, and length validated; amplification and deep nesting rejected.
Codec CVE exposure Each transfer syntax mapped to its decoder and that decoder's known CVEs.
Crypto posture audit in development Flags cleartext transport and data left exposed at the point of use.
Content disarm and reconstruction beta Rebuilds a clean, clinically equivalent file, which neutralizes even unknown attacks.
